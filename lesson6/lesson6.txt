Тест неравномерности АЧХ


Измерение неравномерности. Должна быть возможность изменения допуска. 

Входные данные :
Пользователь может указать свой фильтр пч
пользователь может устанавливать нужные ему диапазоны
- Попробуйте сделать что то типа конфиг файла.

Включить прибор. Установить: 

• Attenuator control – вкл, Аттенюатор — 10дБ 
• 
• фильтр ПЧ 1 кГц 
- установить Код ЦАП.  АЧХ SERV:RFCTL:POW:DAC 6554

• кол-во точек в два раза чаще, чем при заводской калибровке :
 SERVice:SWEep:FREQuency:FACTory устанавливает определенный сегментный план, вам нужно
получить его из SNVNA и увеличить колличество точек для каждого сегмента в 2 раза 

• Включить трассы T11, R11, S11, … и так для всех портов (Делать измерения последовательно или все вместе, как удобно) 

• Дождаться пока пройдет росчерк трасс. достаточно однократного росчерка. 

- Реализовать измерение неравномерности в диапазонах, это можно сделать использовав маркеры, и статистику между маркерами

• Измерить неравномерность АЧХ каждой трассы, это p-p в статистикие(никаких экпаненциальных чисел)

• Выбрать максимальную неравномерность из абсолютных трасс (T и R), и максимальную неравномерность из относительных трасс(S). 

- Занести данные в таблицу. пример в таблице table.png, в шапку таблици необходимо добавить информацию о приборе и программы SNVNA

- перед выполнением задания вам неообходимо договорится о структуре файла json с результатами.
 - теперь будем работать так же в новой ветке(название сами придумайте), и в одном файле, должен быть блок main который вызывает функции
read_config() читает файл config.txt 
start_frequency_test(), результат работы функции файл freq_test.json
generate_table() генерирует html документ из файла json
